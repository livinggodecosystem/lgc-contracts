## Reporting a Vulnerability

The Living God Ecosystem takes security seriously.

If you discover a security vulnerability in any smart contract within this repository, please report it responsibly before making any public disclosure.

When submitting a report, please include:

* A clear description of the issue
* Steps to reproduce the vulnerability
* Potential impact
* Suggested mitigation (if available)

---

## Responsible Disclosure

Please allow adequate time for investigation and remediation before publicly disclosing any security issue.

Responsible disclosure helps protect users, ecosystem participants, and the integrity of the Living God Ecosystem.

---

## Security Practices

The smart contracts in this repository are developed with security as a primary objective and include:

* OpenZeppelin audited libraries
* Ownable access control
* Reentrancy protection
* Emergency pause mechanism
* Safe ERC20 token handling
* Custom Solidity errors
* Extensive unit testing
* NatSpec documentation
